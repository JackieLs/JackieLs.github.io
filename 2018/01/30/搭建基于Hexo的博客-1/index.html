搭建基于 Hexo 的博客 (1)

2018-01-30 • 分类: github博客搭建 • 标签: blog, hexo

直到有一天我发现我干了两年前端，却没留下一点痕迹的时候我才知道，我该写博客了。

————题记

我先说几点我认为写博客的重要意义:

  • 沉淀，总结业务中的经验教训
  • 加深对知识的理解
  • 把自己的认识分享给大家，同时接受大家的批评指正

如果认同这几个观点，请继续往下看，不认同的，请Cmd + Q 或者 Alt + F4。

这些天百度了一下如何搭建一个 github 博客，其中比较流行的有 Hexo、Jekyll，因为 Jekyll 是基于 RubyGem 的，而因为我比较熟悉 NodeJS，所以选择基于NodeJS 的Hexo。

创建文章

官网的命令是 hexo new [layout] title，我们上面的 layout 省略掉了，自动使用的就是 post 模板，配置项是_config.yml 的default_layout: post，这个 post 指的就是 scaffolds -> post.md下面的 post.md 文件。生成的文件在 source -> _posts 下面。点击之后就可以任意编辑了。

生成网站

可以添加 -d 生成静态文件后直接部署，这一步的目的是把所有的 md 以及 theme 的样式、字体等生成 html,css,js，然后放到一个叫做 .deploy_git 的文件夹下面，这个 .deploy_git 就是你的 Nancy.gihub.io 这个 Repository，下一步就是把它推到 github 上
