协商缓存和强制缓存的基本知识

2018-03-02 • 分类: 大前端 • 标签: javascript, nodejs

我们这种小公司是不准求缓存的，但是金鳞岂是池中物，我要追求完美。

强制缓存

啥叫强制？

在 size 列里面出现了大量的(form memory cache)。翻译成中文就是从内存缓存中拿来的数据。这部分状态是 200，这就是强制缓存。那么浏览器是怎么知道我要强制缓存的呢？看下图

当从浏览器请求到一个可缓存的资源的时候，这个资源的 header 都会带上 expires 和 cache-controll:max-age=999999 这样两个字段。

名称 作用 优先级
Expires 规定该资源的最后过期时间，如果超过这个时间，则浏览器会重新向服务器请求该资源 低
max-age 规定该资源缓存的相对时间 高

  • Expires 是浏览器在第一次请求该资源的时候服务器放到 response 的 header 里面，等浏览器第二次请求该资源的时候，如果没有超过 Expires 的时间点，那么就会自动读取缓存，否则就重新请求。Expires 是较老的强缓存管理 header，由于它是服务器返回的一个绝对时间，在服务器时间与客户端时间相差较大时，缓存管理容易出现问题，比如随意修改下客户端时间，就能影响缓存命中的结果。

  • max-age 是浏览器在第一次请求该资源的时候服务器放到 response 的 header 里面，浏览器会记录下这个 request 的时间点，等第二次请求该资源的时候，会用(currentTime - requiestTime) 与 max-age 进行比较，如果大于该事件，那么会重新请求资源。

啥叫协商？

Last-Modified & If-Modify-Since

Request Header

Response Header

  • Last-Modified 是浏览器在第一次请求该资源的时候服务器放到 response 的 header 里面，浏览器会记录这个 header 等浏览器第二次请求该资源的时候，浏览器会在 header 里面加上一个 If-Modify-Since，这个就是上次请求存储的 Last-Modified 的时间，如果这个时间内没有修改，那么就直接返回 304，否则直接返回新的资源以及新的 Last-Modified，重新进行记录。

E-Tag & If-Not-Match

Request Header

Response Header

  • ETag 是浏览器在第一次请求该资源的时候服务器放到 response 的 header 里面，浏览器会记录这个 header，等浏览器第二次请求该资源的时候，浏览器会在 header 里面加上一个 If-None-Match 的 header，把这个与服务器资源的 Etag 进行对比，如果一致则返回 304，读取缓存，否则重新加载资源。每次 response 都会得到一个新的 ETag，无论这个 ETag 是否与本地相同，都会覆盖本地的存储，下次请求使用新的 ETag 进行请求。

两种协商缓存方式的关系

首先服务端记录 Last-Modified 的时间无疑是最方便的，但是在某种情况下，Last-Modified 可能不会变化，这样就导致了无法进行资源更新。所以这时候就会去检查 ETag，ETag 相当于对文件进行 hash，如果对比两个 hash 不一样，那么就代表资源已更新，就会直接下载资源。ETag 是对 Last-Modified 强有力的补充。任何一个都可以单独存在，在不同的场景下会有不同的使用方式。分布式的系统不要添加 ETag，因为每个机器上的 ETag 都不一样。

协商缓存配合强制缓存

在性能优化中，几乎所有的静态资源都会启用强制缓存，部分在一段时间内会更新的数据会启动协商缓存。强制缓存的优先级最高，如果未命中则会去进行协商缓存，仍未命中则请求新资源。
