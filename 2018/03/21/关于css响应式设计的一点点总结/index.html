关于 css 响应式设计的一点点总结

2018-03-21 • 分类: 大前端 • 标签: css, 响应式

在很久以前，有很多响应式的样式框架，比如 Bootstrap，Materializecss。他们都是响应式布局很好的解决方案，至今还有很多项目还在使用。在日常工作中，我也使用过现成的框架，但是框架只能提供一个大体上的有很多实际问题还是解决不了。今天总结一下我在工作中实际遇到的响应式方案的解决方式。

媒体查询

奇弩公司的官网项目中引入了 Bootstrap，其中 Bootstrap 中控制 lg，xs，sm，md 等尺寸的方式就是通过媒体查询来控制的。这种方式可以使浏览器在不同的屏幕尺寸下分别显示不同的样式

宽度大于 991px ，container 的背景就是蓝的，小于就是红的。

REM 布局

之前做过一个页面，这个页面运行在一个 app 里面，这个 app 就是一个浏览器。需要适应各个尺寸下各个屏幕的样式，设计师有个要求，所有的页面大小必须等比例缩放。所以我整体都采用了 rem 布局的方式来实现这个设计要求。

Flex 布局

我是在奇弩的 IDE 上使用这个布局的，这个 IDE 是一款类似于 sublime 的编辑器，这款编辑器有这样一个场景，有 5 个元素要横向排列，其中最中间的一个元素要根据屏幕的宽度进行自适应，有个最小宽度，其余几个元素不变，并且每个元素与每个元素之间的距离要相等，垂直居中。我当时听到这个需求的时候，很不以为意，做起来才发现，用传统的什么 margin，百分比之类的方式是要多费劲有多费劲。于是重新修改了布局方式，采用了 flex 布局，因为是基于 electron 的所以不需要考虑兼容性的问题。针对上面的需求，我给出了以下模拟代码。

container 上的属性

  • display: flex // 触发 flex 布局
  • justify-content: space-between;// 使用各个元素中间间隔都一样的内容布局方式。还有 flex-start,flex-end, space-around，center 等方式。
  • align-items: center; // 水平居中，还有 flex-start,flex-end,strench,baseline
  • flex-direction: row; // 排列方向，还有 column, column-reverse, row-reverse
  • flex-wrap: nowrap; // 多个内容是否换行。
  • flex-flow: row nowrap; //direction 和 wrap 的结合。

content 上的属性

  • flex-shrink: 缩小，default 1 // 等比缩小，禁止负数
  • flex-grow: 放大比例，default 0 // 不放大，禁止负数
  • flex-basis: 初始大小

这三个属性可以写成flex: 1 0 350px;

关于这个比例如何计算的，请看 详解 flex-grow 与 flex-shrink
其中 grow 比较简单，就是剩余空间按比例分，如果加一块小于 1，那么就按自己设置的小数的百分比分，shrink 比较复杂，可以在上文中看到详细计算公式。

flex 布局在 pc 端的兼容不是很好，其实主要是 IE。而在手机端兼容效果却是出奇的好，因为大多数手机浏览器的内核都是 webkit 的。所以在手机端 flex 布局也是经常使用。

总结

实际的工作中，往往都是有很多种方案一起选择的，由媒体查询控制模块的显示与隐藏，由 flex 实现整体的 layout，或者全部使用 rem 来书写整个站的 css。具体选择哪种方案，应该从各个角度去考虑，比如 PC 端，使用百分比 + 媒体查询的方式布局就可以，移动端如果是流式布局使用百分比也可以，一些特殊的就可以用 rem，和 flex 来解决。总之希望大家在实际工作中多多体会哪种方案更加适合哪个项目。
