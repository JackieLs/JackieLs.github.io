JS 的面向对象编程——继承

2018-02-08 • 分类: 大前端 • 标签: javascript, nodejs, 面向对象

之前讲过原型链，JS 的继承就是通过原型链来继承的，不像 java 那样有 class 可以 extends，今天我们好好研究一下 js 是怎么继承的。

什么叫继承

  • 子类可以使用父类的方法 / 属性
  • 子类可以继续定义自己的方法属性
  • 子类可以通过 getter 方式来调用父类的私有属性

JS 中的继承

JS 中的类

在 JS 中是通过构造函数来实现类的概念

JS 中的六种继承方式

一、基本原型链继承

实现
优点
  • 简单
缺点
  • 在子类实例修改上父类函数内部的 引用类型属性 会被提现到每个实例上，因为这类属性都是内存地址的传递。
  • 如果想通过子类的实例去向父类的构造函数传递初始参数耦合性会非常高。

二、借用构造函数

实现
优点
  • 弥补了上面的缺点
缺点
  • 无法实现函数 / 公共属性复用，每个子类实例都持有一个新的 foo/subObj/superObj，太多了就会影响性能，内存爆炸。
  • 无法继承父类原型链上的方法和属性

三、组合继承

最常用的一种继承方式

实现
优点
  • 弥补了上面的缺点
缺点
  • 不能算是缺点，算不够完美吧，因为在每次 new Sub() 的时候，都会调用一次 Super.apply(this, arguments)，而new Super() 已经调用过一次了。还是有内存上的浪费。

这里说明一下，什么时候用构造函数内部属性，什么时候用原型属性，当属性为 非共享属性的时候 在构造函数内部进行声明，并且保证 每次创建实例的时候都会执行一次该构造函数 ，当属性为 共享属性的时候 将属性放在原型链上，供所有实例使用，一旦某实例修改共享属性，则其他实例也会受到影响。

四、原型式继承

这种方式不是很常见，借助原型可以基于已有的 对象 创建新对象，同时还不必因此创建自定义类型，不依赖构造函数。

实现
优点
  • 不需要指定各种构造函数的类型，只定义原型对象即可。
不足
  • 相当于都是通过 prototype 继承的，所有的引用类型的属性都是公共的，这时候其中一个实例修改了这个属性，别的实例也会被修改。

五、寄生式继承

实现
优点
  • 在主要考虑对象而不是自定义类型和构造函数的情况下，寄生式继承也是一种有用的模式。
不足
  • 跟构造函数一样，函数不能复用，用多了内存爆棚。

六、寄生组合式继承
这种是很经典近乎完美的模式，babel 编译 ES6class A extends B的时候使用了这种模式。

实现
优点
  • 高效，SuperType 这个构造函数只被调用了一次。
缺点
  • 有点复杂？我觉得还好。

总结

介绍了六种继承的方式，大同小异，都是在原型链上动手脚。只是在不断地优化效率。现在都是用 ES6 了，babel 直接帮我们把最优化最规范的方式写出来了，不过我们还是要明白其中的原理，说不定有一天我们需要自己去实现一些东西呢。
